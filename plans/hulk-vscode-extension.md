# Plan: HULK VS Code Syntax Highlighter

## Objective
Create a standalone VS Code extension for the HULK programming language with full syntax highlighting, snippets, and language configuration.

## 1. Extension Structure
Create a new directory `vscode/` in the project root with the following structure:
- `vscode/package.json`: Extension manifest.
- `vscode/language-configuration.json`: Language rules (brackets, comments, folding).
- `vscode/syntaxes/hulk.tmLanguage.json`: TextMate grammar for HULK.
- `vscode/snippets/hulk.code-snippets`: Code snippets for common constructs.
- `vscode/README.md`: Basic documentation.
- `vscode/.vscodeignore`: Files to exclude from the package.

## 2. Manifest (`package.json`)
Configure the extension with:
- **Name:** `hulk-vscode`
- **DisplayName:** `HULK Programming Language`
- **Description:** `Syntax highlighting and support for the HULK programming language.`
- **Version:** `0.1.0`
- **Publisher:** `apiad`
- **Author:** `Alejandro Piad Morffis <apiad@apiad.net>`
- **Contributes:**
    - `languages`: `id: hulk`, `extensions: [.hulk]`, `configuration: ./language-configuration.json`
    - `grammars`: `language: hulk`, `scopeName: source.hulk`, `path: ./syntaxes/hulk.tmLanguage.json`
    - `snippets`: `language: hulk`, `path: ./snippets/hulk.code-snippets`

## 3. Syntax Grammar (`hulk.tmLanguage.json`)
The grammar will target the full HULK specification:
- **Keywords:** `let`, `in`, `if`, `else`, `elif`, `while`, `for`, `function`, `type`, `inherits`, `new`, `is`, `as`, `protocol`, `extends`, `def`, `match`, `case`, `default`.
- **Built-in Constants:** `PI`, `E`, `true`, `false`.
- **Built-in Functions:** `print`, `sin`, `cos`, `sqrt`, `exp`, `log`, `rand`, `range`.
- **Operators:** `+`, `-`, `*`, `/`, `^`, `@`, `@@`, `:=`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `&`, `|`, `!`, `=>`.
- **Macro Symbols:** Support for `*` (variable arguments), `@` (symbolic arguments), and `$` (variable placeholders) inside `def` macros.
- **Literals:** Numbers (32-bit floats), Strings (with escape sequences `\"`, `\n`, `\t`).
- **Comments:** Single-line comments starting with `//`.

## 4. Language Configuration (`language-configuration.json`)
- **Comments:** `lineComment: "//"`.
- **Brackets:** `[[ "{", "}" ], [ "[", "]" ], [ "(", ")" ]]`.
- **Auto-closing pairs:** Brackets and quotes.
- **Folding:** Based on markers `{` and `}`.

## 5. Snippets (`hulk.code-snippets`)
Provide snippets for:
- `function`: `function ${1:name}(${2:args}) => ${3:expression};`
- `type`: `type ${1:Name} {\n\t$0\n}`
- `let`: `let ${1:var} = ${2:value} in $0`
- `if`: `if (${1:condition}) $2 else $3`
- `def`: `def ${1:macro}(${2:args}) => $0`

## 6. Makefile Integration
Add a target to the root `makefile`:
```makefile
.PHONY: vscode
vscode:
	cd vscode && npx @vscode/vsce package -o ../hulk-vscode.vsix
```

## 7. Registration and Publishing
To register as a publisher:
1.  **Create a Microsoft Account** (if you don't have one).
2.  **Sign in to the Visual Studio Marketplace Management Portal:** [https://marketplace.visualstudio.com/manage](https://marketplace.visualstudio.com/manage).
3.  **Create a Publisher:** Use the ID `alejandro-piad` and fill in your details.
4.  **Personal Access Token (PAT):** Generate a PAT from Azure DevOps with "Marketplace (Publish)" permissions.
5.  **Publish via CLI:**
    - Run `npx @vscode/vsce login alejandro-piad` and enter your PAT.
    - Run `npx @vscode/vsce publish` from the `vscode/` directory.

Alternatively, you can manually upload the `.vsix` file generated by the `makefile` through the Management Portal.
